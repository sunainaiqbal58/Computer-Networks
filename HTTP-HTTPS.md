# 🌐 HTTP and HTTPS

## Overview

HTTP (Hypertext Transfer Protocol) and HTTPS (Hypertext Transfer Protocol Secure) are communication protocols used to transfer data between a web browser and a web server.

Whenever we visit a website, the browser uses one of these protocols to request and receive web pages.

---

## HTTP

HTTP is the standard protocol used to access websites. It transfers data in plain text, which means the information can be intercepted by attackers if the connection is not protected.

**Default Port:** 80

---

## HTTPS

HTTPS is the secure version of HTTP. It encrypts the communication between the browser and the web server using SSL/TLS, making it much safer for users.

**Default Port:** 443

---

## HTTP vs HTTPS

| HTTP | HTTPS |
|------|-------|
| Data is not encrypted | Data is encrypted |
| Uses Port 80 | Uses Port 443 |
| Less secure | More secure |
| Suitable for public information | Suitable for login pages, banking, and online payments |

---

## Why HTTPS is Important

- Protects sensitive information during transmission.
- Prevents attackers from reading or modifying data.
- Verifies the identity of the website through digital certificates.
- Builds trust by displaying the padlock icon in the browser.

---

## Cyber Security Relevance

HTTPS plays an important role in protecting users from data theft and man-in-the-middle attacks. Security professionals always recommend using HTTPS instead of HTTP, especially when handling passwords, personal information, or financial transactions.

---

## Quick Notes

- HTTP uses **TCP Port 80**.
- HTTPS uses **TCP Port 443**.
- HTTPS uses **SSL/TLS** encryption.
- Most modern websites use HTTPS by default.

---

## Summary

HTTP and HTTPS are essential web protocols used for communication between browsers and web servers. HTTPS provides secure and encrypted communication, making it the preferred protocol for modern websites.
